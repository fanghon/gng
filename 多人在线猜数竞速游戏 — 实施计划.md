# 多人在线猜数竞速游戏 — 实施计划

## Context

现有 `caishu.html` 是一个单人网页猜数游戏（1~100，有大小提示和范围缩小）。目标：改造成多人在线竞速模式 — 多人同时猜同一个数字，共享缩小范围，谁先猜中谁赢，有跨回合排行榜。

## 技术选型

- **后端**: Node.js + Express（静态文件）+ `ws`（WebSocket）
- **前端**: 单 HTML 文件，内嵌 CSS + JS，通过 WebSocket 与服务器实时通信
- **依赖**: `express`, `ws`（仅两个依赖）

## 核心机制

### 共享范围
所有玩家共享同一个缩小范围。任何人猜错，范围对所有人收紧。例如 A 猜 50 太小 → 范围变为 [51,100]，B 也看到这个范围。这是竞速的核心：别人的猜测为你提供信息。

### 房间系统
- 创建房间 → 获得 6 位房间码（如 `A3F9K2`）
- 输入房间码加入
- 房主点"开始游戏"（需 ≥2 人）
- 房主可开下一轮

### 排行榜
每轮胜者 +1 胜，房间内跨回合累计。

## 文件详解

### 1. `server.js`

```
数据：rooms = Map<code, Room>
Room: { code, hostId, players: Map<id, Player>, state, answer, range, roundNumber }
Player: { id, name, ws, wins, attempts, finished, lastGuessTime }

消息协议 (Client → Server):
  setName      { name }
  createRoom
  joinRoom     { code }
  startGame
  guess        { value }
  nextRound
  leaveRoom

消息协议 (Server → Client):
  roomCreated   { code, playerId }
  roomJoined    { code, playerId, players, hostId }
  playerJoined  { player }
  playerLeft    { playerId, newHostId? }
  playerList    { players, hostId }
  gameStarted   { range, roundNumber }
  guessResult   { playerId, playerName, guess, hint, range, attemptCount }
  roundEnd      { winner, answer, leaderboard }
  error         { message }

关键逻辑:
- 猜数服务端校验（范围、状态、防连点 200ms）
- 断线处理：移除玩家、转移房主、空房清理
- 房间码生成：排除易混字符(0/O, 1/I)
```

### 2. `public/index.html`

4 个屏幕（div 切换显示）:

| 屏幕 | 内容 |
|------|------|
| 入口 | 输入昵称 + 创建/加入房间 |
| 大厅 | 房间码（可复制）、玩家列表、开始按钮 |
| 游戏 | 输入框、提示、共享范围、玩家状态、动态流 |
| 结算 | 胜者、答案、排行榜、下一轮/退出 |

UI 保留原版的渐变背景、白色卡片、圆角、消息颜色（橙=偏小、蓝=偏大、绿=猜中）。

## 游戏流程

```
1. A 创建房间 → 得到房间码
2. B 输入房间码加入 → 双方大厅看到彼此
3. A 点开始 → 服务端生成答案，广播 gameStarted
4. A 猜 50 → 太小 → 广播给所有人，范围收紧
5. B 猜 75 → 太大 → 广播
6. B 猜 67 → 正确！→ 广播 roundEnd，B 胜场+1
7. A 点下一轮 → 新答案，roundNumber+1
```

## 验证方式

1. `npm install && node server.js`
2. 打开两个浏览器窗口访问 `http://localhost:3000`
3. 窗口 A 创建房间，窗口 B 加入
4. 测试竞速猜数，验证范围共享、胜负判定、排行榜累计
